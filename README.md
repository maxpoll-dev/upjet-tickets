# Намеренные допущения

- Список основных пунктов где сделал намеренное допущение и упрощение. 
- Цель показать что не слепо пишу промпты и нажимаю апрув.

## Архитектура

### Docker

1) Не стал делать фронт внутри laravel через их решение Inertia. Раздельное удобнее, проще гонять через CI/CD и нет пересечений в работе фронтедера с бекендером.
2) Так же не стал использовать caddy в контейнере бека для проксирования. Вынес это на отдельный nginx.

## Backend

### Таблицы

1) По хорошему надо было создать отдельную таблицу транзакции, что бы разграничить с заказами и была возможность делать возврат.
2) Вырезал юзеров и сессии. Первоначально была идея реализовать привязку заказа к сессии(куки), но передумал в сторону токена.
3) Нет данных платежной системы в заказе, так как реализовал фейк оплату.

### Код

1) Сделал фейк страйп подобной оплаты с отправкой данных на вебхук.
2) Реализовал только один сценарий - Оплачено в обработке вебхука.
3) Не сделал юнит тесты
4) Минуя фабрику сделал наполнение через сидер. 
5) Снес лишние папки и файлы. Возможно в экосистеме laravel принято все это оставлять.

## Frontend

1) Меньше затратил времени на реализацию. (очень много потратил на изучение доки laravel 13 - orm, архитектура и классы/свойства/методы)
2) Реализовал с ui библиотекой и минимум инструментов из экосистемы реакт.
